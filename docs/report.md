## 01-10：基础算法与数据结构

| # | 题目 | 核心实现 |
|---|------|----------|
| 01 | 插入排序 | 降序插入排序，从第二个元素开始向左比较交换 |
| 02 | 归并排序 | 分治递归 + 临时数组合并，降序排列 |
| 03 | 快速排序 | Hoare 分区，选中间元素为 pivot，降序排列 |
| 04 | 线性查找 | `for` 循环逐个比较 `strcmp` |
| 05 | 折半查找 | 已按字母排序的数组上二分查找 |
| 06 | 栈迷宫 (DFS) | 递归深度优先搜索，标记已访问，回溯找路径 |
| 07 | 队列迷宫 (BFS) | 队列广度优先搜索，`prev` 数组回溯最短路径 |
| 08 | 环形队列约瑟夫环 | 环形队列模拟报数淘汰，50 人报 5，幸存 19 号 |
| 09 | 单词计数器 | 状态机法：`inWord` 标志统计非空格序列 |
| 10 | `my_strcpy` | 逐字符拷贝到目标数组，补 `\0` |

### 要点
- 02/03 使用全局 `Student` 数组，通过成绩字段降序
- 06/07 同一迷宫，DFS 输出 13 步路径，BFS 输出 9 步最短路径
- 08 输出格式要求：`淘汰: X` + `最后剩下的人是: X`
- 09 是空格分词，`Don't` 和 `you,` 各算一个单词（共 18）

## 21-30：高级数据结构与内联汇编

| # | 题目 | 核心实现 |
|---|------|----------|
| 21 | 单链表约瑟夫环 | `make_node`/`insert`/`delete`/`traverse`，`next_wrap` 环形遍历 |
| 22 | 双向循环队列 | 哨兵节点双向链表，`insert` 头插，`delete` 尾删 |
| 23 | 环形链表约瑟夫环 | 尾节点 `next` 指向头，`prev` 辅助删除 |
| 24 | 二叉树前序遍历 | 层次建树（队列），递归 + 迭代（栈）前序 |
| 25 | BST 字母统计 | 二叉搜索树插入/更新字母频次，中序输出 |
| 26 | Hash 词频统计 | djb2 哈希 + 链地址法 + `qsort` 降序排列 |
| 27 | 内联汇编 GCD | `div` 指令：`xor edx,edx; div ebx; mov ebx,eax; mov edx,ebx` |
| 28 | 运算溢出检测 | `add/setc`、`sub/setc`、`mull/edx`、`test/setz` |
| 29 | 字节序转换 | 位运算 `(n & 0xFF) << 24 | ...` 交换字节 |
| 30 | DEBUG_PRINT 宏 | 条件编译 `#if DEBUG_LEVEL >= 2`，`__func__` `__LINE__` |

### 要点
- 21/23 约瑟夫环三种实现（单链表、双向循环、环形）各有不同删除策略
- 24 需要用 `INT_MIN` 表示空节点，队列辅助层次建树
- 27 汇编约束需用 `"=a"(result)` 而非 `"=r"(result)` 确保 eax 连接
- 28 `mull` 结果在 `edx:eax`，通过 `setne` 检测高 32 位
- 30 Makefile 需添加 `-DDEBUG_LEVEL=2`

## 31-40：工程化与并发编程

| # | 题目 | 核心实现 |
|---|------|----------|
| 31 | 事件处理器 | 函数指针数组 `g_callbacks[EVENT_MAX]` + 参数数组 |
| 32 | `container_of` | `(type *)((char *)(ptr) - offsetof(type, member))` |
| 33 | 动态数组 GArray | 初始容量 16，满时 2 倍 `realloc`，泛型 `memcpy` |
| 34 | 协议头解析器 | `#pragma pack(1)` 位域结构体 + 网络序转换 |
| 35 | ELF 信息查看 | 解析 `/bin/ls` 的 ELF 头 + 程序头表，大/小端适配 |
| 36 | LRU 缓存 | 哈希表 O(1) 查找 + 双向链表 O(1) 淘汰 |
| 37 | 位图操作 | `bitmap[bit/8] \|= (1 << (bit%8))` |
| 38 | 线程安全环形缓冲 | `pthread_mutex_t` + `pthread_cond_t` 生产者消费者 |
| 39 | `strtok_r` | 跳过前导分隔符 + 标记开始 + 查找结束 + 置 `\0` |
| 40 | Bloom 过滤器 | 3 个哈希函数 + 位图插入/查询，允许误判 |

### 要点
- 32 必须使用 `-std=gnu11` 或默认 GNU 扩展（`typeof`）
- 34 大端 16 位转换 `((be >> 8) | (be << 8))`
- 35 需处理文件与主机端序不一致情况，逐字段 `bswap`
- 36 LRU 需同步维护哈希表索引和链表顺序
- 38 需要 `-pthread` 链接，Makefile 需添加编译标志
- 40 三个哈希：`sum(c) % 100`、`sum(c*2) % 100`、`sum(c*3) % 100`
